*1. Інструкція запуску:*
1) Створення проекту на WebStorm;
   
2) Встановлення бібліотек: npx expo install firebase;  npx expo install @react-native-async-storage/async-storage; npx expo install expo-router react-native-safe-area-context react-native-screens expo-linking expo-constants expo-status-bar
 
3) Запуск серверу командою: npx expo start
 
4) Підключення через телефон за допомогою програми Expo GO
   
*2. Опис реалізованого функціоналу:*

Реєстрація та вхід; збереження додаткової інформації: ім’я, місто, вік; редагування особистої інформації; видалення акаунта; зміна пароля; відновлення пароля; вихід з акаунта;
дані захищені на рівні сервера за допомогою Firestore Security Rules.

*4. Висновки:*

Під час виконання лабораторної роботи я опанував навички інтеграції мобільного додатка на базі React Native із хмарною платформою Firebase.
Вдалося реалізувати надійну систему автентифікації, яка використовує токени, що зберігаються локально через бібліотеку AsyncStorage. 
Це забезпечує стабільну підтримку сесії навіть після повного закриття додатка. У процесі розробки детально вивчив роботу з NoSQL базою даних Firestore, 
зокрема принципи створення та структурування документів із прямою прив'язкою до унікального ідентифікатора користувача.
Окрему увагу було приділено безпеці: я на практиці застосував механізми захисту даних на стороні сервера через Firestore Security Rules, 
що повністю виключає можливість несанкціонованого доступу сторонніх осіб до чужих профілів. Крім того, налаштував динамічну навігацію за допомогою Expo Router, 
яка в автоматичному режимі реагує на будь-які зміни стану авторизації, забезпечуючи правильний редирект між екранами входу та основним інтерфейсом додатка
