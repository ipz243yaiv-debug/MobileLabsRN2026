*1. Інструкція запуску:*

1) Створення проекту на WebStorm;
   
2) Встановлення бібліотек: npx expo install expo-router react-native-safe-area-context react-native-screens expo-linking expo-constants expo-status-bar
 
3) Запуск серверу командою: npx expo start
 
4) Підключення через телефон за допомогою програми Expo GO
   
*2. Опис реалізованого функціоналу:*

Реєстрація, вхід та вихід із системи, дані чи зареєстрований користувач, каталог товарів, перегляд інформації про певний товар;

Захищені маршрути: Використано групу (app) з перевіркою авторизації. Якщо користувач не увійшов — система автоматично редиректить його на логін

Динамічні сторінки: Реалізовано перехід на сторінку конкретного товару за його id. Дані підтягуються динамічно через useLocalSearchParams.

Навігація: Використано Expo Router з підтримкою груп маршрутів та кастомних заголовків у Stack.

Обробка помилок: Додано екран +not-found для переходів за неіснучими посиланнями.

*4. Висновки:*

1. Яким чином за допомогою Expo Router реалізується перенаправлення неавторизованого користувача?

Через спеціальний файл макета _layout.jsx. У ньому перевіряється стан авторизації. Якщо вона false, ми просто повертаємо компонент <Redirect href="/login" />. 
Як тільки користувач намагається зайти в папку, макет бачить, що входу не було, і "викидає" його на сторінку логіна.

2. У чому полягає різниця між використанням компонента <Link> та метода router.push()?
<Link> — це як звичайна силка в браузері, використовується в інтерфейсі.Router.push() спрацьовує після певної дії, наприклад після натискання на кнопку "Увійти".
   
3. Як працюють динамічні маршрути в Expo Router і як отримати передані параметри?
Створюються за допомогою квадратних дужок у назві файлу. Expo Router розуміє, що замість [назва] у назві може бути будь-яке значення.
Щоб отримати цей параметр всередині екрана, використовується хук useLocalSearchParams(). Він повертає об'єкт, з якого просто витягується потрібне значення.

4. Чому стан авторизації доцільно зберігати у глобальному контексті (React Context), а не в локальному стані компонента?
Якщо тримати стан у звичайному useState всередині одного екрана, то інші екрани про нього нічого не знатимуть.

5. Для чого використовуються групи маршрутів (folderName) і як вони впливають на URL-адресу?
Для організації коду, вони не відображаються в URL-адресі, що дає змогу розділити додаток на логічні частини.
